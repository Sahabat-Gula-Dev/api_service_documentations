Information Service

Endpoint-endpoint ini digunakan untuk mengelola data informasi singkat yang dapat ditampilkan secara acak di aplikasi.

Informations

POST /informations

Tambah Informasi Baru

Menambahkan data informasi baru ke dalam database. Endpoint ini memerlukan otorisasi. Minimal salah satu dari heading atau content harus diisi.

Authorization: Bearer Token

Request Body application/json

Key Tipe Deskripsi
heading string Judul atau kalimat pembuka informasi. Opsional.
content string Isi konten dari informasi. Opsional.

Success Response 201 Created

{
    "status": "success",
    "message": "Informasi berhasil ditambahkan.",
    "data": {
        "information": {
            "id": 1,
            "heading": "Tahukah Kamu?",
            "content": "Minum air putih yang cukup dapat membantu menjaga kadar gula darah tetap stabil.",
            "created_at": "2025-09-20T17:30:00.000Z"
        }
    }
}
GET /informations

Ambil Informasi Acak

Mengambil satu data informasi secara acak dari database. Endpoint ini bersifat publik.

Authorization: None

Success Response 200 OK

{
    "status": "success",
    "data": {
        "information": {
            "id": 1,
            "heading": "Tahukah Kamu?",
            "content": "Minum air putih yang cukup dapat membantu menjaga kadar gula darah tetap stabil.",
            "created_at": "2025-09-20T17:30:00.000Z"
        }
    }
}
GET /informations/{id}

Detail Informasi

Mengambil detail satu data informasi berdasarkan ID-nya.

Authorization: None

Success Response 200 OK

{
    "status": "success",
    "data": {
        "information": {
            "id": 1,
            "heading": "Tahukah Kamu?",
            "content": "Minum air putih yang cukup dapat membantu menjaga kadar gula darah tetap stabil.",
            "created_at": "2025-09-20T17:30:00.000Z"
        }
    }
}
PUT /informations/{id}

Perbarui Informasi

Memperbarui data informasi berdasarkan ID. Semua field bersifat opsional (kirim hanya yang ingin diubah).

Authorization: Bearer Token

Request Body application/json

Key Tipe Deskripsi
heading string Judul baru. Opsional.
content string Konten baru. Opsional.

Success Response 200 OK

{
    "status": "success",
    "message": "Informasi berhasil diperbarui.",
    "data": {
        "information": {
            "id": 1,
            "heading": "Info Kesehatan Terkini",
            "content": "Penelitian terbaru menunjukkan kaitan antara stres dan kenaikan gula darah.",
            "created_at": "2025-09-20T17:30:00.000Z"
        }
    }
}
DELETE /informations/{id}

Hapus Informasi

Menghapus data informasi secara permanen berdasarkan ID.

Authorization: Bearer Token

Success Response 200 OK

{
    "status": "success",
    "message": "Informasi berhasil dihapus."
}
